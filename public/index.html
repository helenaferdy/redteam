Week 3 — Attack Surface Concepts

GOAL — Understand what makes a system vulnerable before you try to exploit it. Map every service and misconfiguration to an attack vector.
01 What Is an Attack Surface? Concept
▸ Goal

Build the mental model: a system is vulnerable because it exposes something reachable with a weakness you can exploit.

▸ ▸ Context

Three layers: Network surface (open ports) → Software surface (apps + versions = CVEs) → Configuration surface (misconfigurations — human errors).

Reachability × Severity = Real Risk. A critical RCE firewalled from your subnet is useless. A medium bug reachable from anywhere is exploitable.
02 Windows Attack Surface Concept
▸ Goal

Map every Windows service from your Week 2 DC scan to an attack category.

▸ ▸ Execute

▸ ▸ Tool Reference

SMB (445)

Most attacked Windows service. File shares, remote service creation, pass-the-hash. If open, you have an initial vector.

WinRM (5985)

PowerShell remoting. Stealthier than SMB — used by real admins daily. Evil-WinRM is the go-to client.

RDP (3389)

Remote Desktop. Weak passwords = full GUI access. Tools: Hydra, Crowbar, xfreerdp.

Local PrivEsc

Unquoted service paths, weak perms, AlwaysInstallElevated, token impersonation (Potato), UAC bypass.

03 Linux Attack Surface Concept
▸ Goal

Map the attack surface of the Ubuntu server (172.24.24.23).

▸ ▸ Execute
  1. From W2 nmap: SSH (22), VulnLab (1337), Samba (445)
  2. Local vectors (need shell):
    find / -perm -4000 -type f 2>/dev/null # SUID find / -writable -type f 2>/dev/null # World-writable cat /etc/crontab && ls -la /etc/cron.* # Cron ls -la /var/run/docker.sock # Docker socket
▸ ▸ Tool Reference

SUID Binaries

Run as owner (often root). Exploit a SUID binary to escalate. Check /usr/bin/passwd, sudo.

Docker Socket

/var/run/docker.sock = Docker API. If writable, create privileged container → root on host.

Cron Jobs

Scheduled tasks, often root. Overwrite world-writable script → code runs as root next tick.

03 Understand NTLM & Kerberos Concept
▸ Goal

Build mental model of Windows authentication. NTLM = challenge-response. Kerberos = ticket-based.

▸ ▸ Context

NTLM

Challenge-response. Server sends challenge → client encrypts with NTLM hash. Hash = credential (pass-the-hash). No central authority.

Kerberos

Ticket-based. KDC on DC issues TGT → TGS. Forge TGT with KRBTGT hash = Golden Ticket. TGS encrypted with service hash → crackable (Kerberoasting).

KRBTGT

KDC's service account. Its NTLM hash is the domain master key. Forge TGTs forever until password rotated twice.

Pass-the-Hash

Use NTLM hash directly, no cleartext needed. Tools: Mimikatz, Impacket psexec.py, CrackMapExec.

Week 6 — Lateral Movement

GOAL — Starting from the compromised workstation, reach the Domain Controller using lateral movement techniques.
01 Understand the Lateral Movement Toolkit Concept
▸ Goal

Build mental model of every protocol and tool used to move between hosts, and the tradeoffs.

▸ ▸ Protocol Reference

SMB (445)

File sharing + remote service creation. Primary lateral movement channel. Always open between domain members.

PsExec

Creates temp service via SMB, runs as SYSTEM. Noisy (creates service). Stealth: low.

WinRM (5985)

PowerShell remoting. Looks like normal admin work. Most OPSEC-safe. Stealth: high.

WMI (135)

Spawns via WmiPrvSE.exe, child process. Medium stealth, no service creation.

Pass-the-Hash

Authenticate with NTLM hash directly. Tools: Mimikatz, Impacket, CrackMapExec.

netexec

Swiss-army knife: spray creds, exec commands, dump SAM/LSA, pivot SMB/WinRM.

Week 8 — Threat Intelligence & Vulnerability Prioritization

01 Understand the Vulnerability Scoring Stack Concept
▸ Goal

Learn CVE, CVSS, EPSS, and CISA KEV — the four standards driving real-world prioritization.

▸ ▸ Scoring Standards

CVE

CVE-YYYY-NNNNN. Unique public vuln ID maintained by MITRE. Look up at nvd.nist.gov.

CVSS

0.0-10.0. Standardized severity: attack vector, complexity, privileges, user interaction, CIA impact.

EPSS

0%-100%. Probability of exploitation in wild within 30 days. Data-driven. >10% = strong signal.

CISA KEV

"Exploited right now" list. Federal agencies must remediate within set deadlines. Any KEV entry = Critical.
